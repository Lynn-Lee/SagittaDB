# Pack C2 功能测试文档 — 实例数据库注册管理

> **测试版本：** SagittaDB v1.0-beta
> **对应计划：** Pack C2 — 实例数据库注册管理
> **测试环境：** http://localhost（前端）/ http://localhost:8000（后端）
> **前置条件：** Sprint 1 测试通过，已创建至少一个可连接的数据库实例
> **测试账号：** admin / Admin@2024!

---

## 测试范围

| 模块 | 覆盖内容 |
|---|---|
| 数据库注册 | 手动添加数据库、从引擎同步 |
| 数据库管理 | 启用/停用、备注编辑 |
| 类型适配 | Oracle 显示 Schema、Redis 显示数据库编号 |
| 权限控制 | 停用库不能提交工单、不能在线查询；普通用户全局不可见 |
| 启停标签 | 管理员下拉框显示"已禁用"标签，普通用户不显示停用库 |

### TC-C2-004a 普通用户不可见停用数据库（API）

| 项目 | 内容 |
|---|---|
| **用例编号** | TC-C2-004a |
| **测试场景** | 普通用户调用数据库列表 API 时不返回 is_active=False 的库 |
| **前置条件** | 已创建普通用户 testuser，已为其分配资源组和查询权限；数据库 mysql 设为 is_active=False |
| **测试步骤** | 1. 以 testuser 身份登录<br>2. 调用 GET /api/v1/instances/{id}/databases/<br>3. 调用 GET /api/v1/instances/{id}/db-list/ |
| **预期结果** | databases 接口仅返回 is_active=True 的库名；db-list 接口仅返回 is_active=True 的注册记录 |
| **实际结果** | ✅ 通过（2026-04-12 验证：testuser 仅返回 testdb，不返回 mysql/information_schema 等禁用库） |
| **状态** | ✅ 通过 |

### TC-C2-004b 管理员可见停用数据库并标灰

| 项目 | 内容 |
|---|---|
| **用例编号** | TC-C2-004b |
| **测试场景** | 管理员可在 API 和前端看到停用库，前端显示"已禁用"标签 |
| **前置条件** | 数据库 mysql 设为 is_active=False |
| **测试步骤** | 1. 以 admin 身份调用 GET /api/v1/instances/{id}/db-list/?include_inactive=True<br>2. 前端各页面（在线查询、查询权限、SQL 优化、数据字典、归档、工单提交）选择实例后查看数据库下拉框 |
| **预期结果** | API 返回包含 is_active=False 的库并标记 is_active 字段；前端下拉框中停用库显示"已禁用"标签（灰色 Tag） |
| **实际结果** | ✅ 通过（2026-04-12 验证：db-list 返回 5 个库含 is_active=false 的系统库；前端 6 个页面均显示"已禁用"标签） |
| **状态** | ✅ 通过 |
